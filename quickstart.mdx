## Prerequisites

<Callout kind="info" title="Before You Begin">
Ensure you have:
- A valid email address for account verification
- Team member emails ready for invitations
- Existing documents in Markdown, PDF, or Google Docs format for import
</Callout>

## Create and Verify Your Account

Follow these steps to set up your Converted account quickly.

<Steps>
  <Step title="Sign Up" icon="user-plus">
    Visit `https://app.converted.co.uk/signup` and enter your email, name, and a strong password.
    
    Click **Create Account**.
  </Step>
  <Step title="Verify Email" icon="mail">
    Check your inbox for the verification email from Converted.
    
    Click the verification link to activate your account.
  </Step>
  <Step title="Log In" icon="log-in">
    Return to the login page and sign in with your verified credentials.
    
    You'll land on the dashboard.
  </Step>
</Steps>

## Invite Team Members and Set Permissions

Collaborate by adding your team with role-based access.

<Callout kind="tip">
Use roles like **Admin** for full access, **Editor** for content changes, and **Viewer** for read-only.
</Callout>

<Tabs>
  <Tab title="Admin Invite" icon="shield">
    1. Go to **Team** > **Members** in the sidebar.
    2. Click **Invite Member**.
    3. Enter the email and select **Admin**.
    
    ```markdown
    Member: john@team.com
    Role: Admin
    ```
  </Tab>
  <Tab title="Editor Invite" icon="edit-3">
    Follow the same steps but select **Editor** role.
    
    Limit editors to specific projects during onboarding.
  </Tab>
  <Tab title="Viewer Invite" icon="eye">
    Select **Viewer** for stakeholders who need to review docs.
    
    Viewers cannot edit or delete content.
  </Tab>
</Tabs>

## Build Your First Documentation Project

Create a new space for your project documentation.

<Steps>
  <Step title="Create Space" icon="plus">
    From the dashboard, click **New Space**.
    
    Name it (e.g., "Project Alpha Docs") and add a description.
  </Step>
  <Step title="Add First Page" icon="file-plus">
    Inside the space, click **New Page**.
    
    Choose **MDX** template for rich documentation.
  </Step>
  <Step title="Edit Content" icon="edit">
    Use the visual editor to add headings, components, and code.
    
    Preview changes in real-time.
  </Step>
</Steps>

<CodeGroup tabs="MDX Example">
  ```mdx
  ## Welcome
  
  Start writing your docs here.
  
  <Callout kind="success">
    Your first page is live!
  </Callout>
  ```
</CodeGroup>

## Navigate the Dashboard and Editor

<Tabs>
  <Tab title="Dashboard" icon="layout">
    - **Spaces**: View all documentation spaces
    - **Team**: Manage members and permissions
    - **Settings**: Configure billing and integrations
    - **Analytics**: Track page views and edits
  </Tab>
  <Tab title="Editor" icon="edit-3">
    Key features:
    
    | Feature       | Description                  |
    |---------------|------------------------------|
    | Visual Editor | Drag-and-drop components     |
    | Preview       | Live rendering of MDX        |
    | Version History | Revert changes easily      |
    | Search        | Find content across spaces   |
  </Tab>
</Tabs>

## Import Existing Documents

Bring in your current docs seamlessly.

<Steps>
  <Step title="Upload Files" icon="upload">
    In your space, select **Import** > **Files**.
    
    Drag and drop Markdown or PDF files.
  </Step>
  <Step title="Google Docs" icon="file-text">
    Connect your Google account via **Import** > **Google Drive**.
    
    Select docs to convert to MDX.
  </Step>
  <Step title="Review and Publish" icon="check-circle">
    Edit imported content and publish.
    
    Converted auto-converts formats where possible.
  </Step>
</Steps>
