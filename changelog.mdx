<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]}>
## New Features

- Initial release of the MDX Documentation Starter Kit
- Full support for 15+ components including Update, Steps, and CodeGroup
- Integrated with Converted's conversion resources and calculators

## Breaking Changes

- Removed legacy H1 heading support; always start with H2
- Updated attribute syntax to require double quotes for strings

## Migration Notes

```bash
# Update your config
npm install converted-docs@1.0.0
```

Review your frontmatter and component usage for compliance.
</Update>

## Migration Guide

If you encounter breaking changes from v1.0.0, follow these steps to update your documentation:

<Steps>
  <Step title="Backup Your Site" icon="database">
    Export your current MDX files to a safe location.
  </Step>
  <Step title="Update Dependencies" icon="package">
    <CodeGroup tabs="npm,yarn">
    ````bash
    npm install converted-docs@latest
    ````
    ````bash
    yarn add converted-docs@latest
    ````
    </CodeGroup>
  </Step>
  <Step title="Scan for H1 Headings" icon="search">
    Replace any `# Heading` with `## Heading` throughout your files.
  </Step>
  <Step title="Test Parsing" icon="play">
    Run your build command and fix any syntax errors.
  </Step>
</Steps>

<Callout kind="alert">
Breaking changes are rare but documented here. Always test in a staging environment before production.
</Callout>
